# WSL2 + Ubuntu + Docker 構築手順とプロキシ設定まとめ

## 構成方針

| 項目 | 方針 |
|---|---|
| 仮想環境 | WSL2 + Ubuntu |
| Docker | Docker Engine をUbuntu内に直接インストール(Docker Desktop不使用) |
| systemd | `/etc/wsl.conf` で有効化(Dockerの自動起動のため推奨) |
| プロキシ | Windows側はPACスクリプト方式 → WSL/Dockerでは**PACは直接使えない**ため、固定プロキシURLに変換して設定する |

---

## セットアップ手順

### Step 1: WSL2 + Ubuntu の導入

1. PowerShell(管理者)で実行:

   ```powershell
   wsl --install
   ```

2. 再起動後、Ubuntuを起動してユーザー名・パスワードを設定

3. バージョン確認(VERSIONが「2」であること):

   ```powershell
   wsl -l -v
   ```

### Step 2: systemd の有効化

`/etc/wsl.conf` に以下を追記:

```ini
[boot]
systemd=true
```

PowerShellで `wsl --shutdown` を実行してWSLを再起動。

### Step 3: プロキシURLの特定(PACスクリプトから)

> **背景**: apt・curl・Dockerデーモン等のLinux CLIツールはPACスクリプトを解釈できないため、PACの中身から固定プロキシURLを取り出して設定する。

1. Windowsのプロキシ設定(設定 → ネットワークとインターネット → プロキシ)で
   「セットアップスクリプトを使う」のスクリプトアドレスを確認

2. PACファイルをダウンロードして中身を確認:

   ```powershell
   Invoke-WebRequest -Uri "http://proxy.example.com/proxy.pac" -OutFile proxy.pac
   ```

3. ファイル内の `return "PROXY ホスト:ポート"` の記述からプロキシURLを抽出
   (複数記載されている場合はフェイルオーバー候補のため、先頭のものを使用)

4. `DIRECT` を返している条件(社内ドメイン等)を `NO_PROXY` の設定値として控える

PACファイルの例:

```javascript
function FindProxyForURL(url, host) {
    if (isPlainHostName(host) || dnsDomainIs(host, ".example.local"))
        return "DIRECT";
    return "PROXY proxy01.example.com:8080; PROXY proxy02.example.com:8080";
}
```

→ この例ではプロキシURLは `http://proxy01.example.com:8080`、
`NO_PROXY` には `localhost,127.0.0.1,.example.local` を設定する。

### Step 4: Ubuntu側プロキシ設定(3層すべてに必要)

#### ① シェル環境変数(curl等のCLIツール用)

`~/.bashrc` または `/etc/profile.d/proxy.sh` に追記:

```bash
export HTTP_PROXY="http://proxy.example.com:8080"
export HTTPS_PROXY="http://proxy.example.com:8080"
export NO_PROXY="localhost,127.0.0.1"
export http_proxy=$HTTP_PROXY
export https_proxy=$HTTPS_PROXY
export no_proxy=$NO_PROXY
```

#### ② apt 用(aptは環境変数を参照しないため別途必要)

`/etc/apt/apt.conf.d/95proxy` を作成:

```
Acquire::http::Proxy "http://proxy.example.com:8080";
Acquire::https::Proxy "http://proxy.example.com:8080";
```

※ 認証付きプロキシの場合は `http://ユーザー:パスワード@proxy.example.com:8080` 形式
(パスワードに記号がある場合はURLエンコードが必要)

#### ③ Docker 用

**デーモン用(docker pull / push)** — `/etc/docker/daemon.json`:

```json
{
  "proxies": {
    "http-proxy": "http://proxy.example.com:8080",
    "https-proxy": "http://proxy.example.com:8080",
    "no-proxy": "localhost,127.0.0.1"
  }
}
```

設定後:

```bash
sudo systemctl restart docker
```

**コンテナ/ビルド用(docker run / build)** — `~/.docker/config.json`:

```json
{
  "proxies": {
    "default": {
      "httpProxy": "http://proxy.example.com:8080",
      "httpsProxy": "http://proxy.example.com:8080",
      "noProxy": "localhost,127.0.0.1"
    }
  }
}
```

確認:

```bash
docker info | grep -i proxy
```

### Step 5: Docker Engine のインストール

Step 4の①②が済んでいればプロキシ経由でaptが使える。

```bash
sudo apt update
sudo apt install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | sudo tee /etc/apt/sources.list.d/docker.list
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

sudoなしで使えるように設定(要再ログイン):

```bash
sudo usermod -aG docker $USER
```

動作確認:

```bash
docker run hello-world
```

## 主要な注意点

### プロキシ関連

- **プロキシ設定は3層**(シェル / apt / Docker)それぞれ独立している。
  「pullは成功するがbuild中のapt installが失敗する」等の症状は設定漏れの層を疑う
- **PACが複雑な場合**(複数プロキシの使い分け、NTLM/Kerberos認証あり)は
  固定URL方式では不十分なことがある
  → Windows側にPAC対応中継プロキシ(**px**、**alpaca** 等)を立て、
  WSLからはそれを固定プロキシとして利用する

  ```
  WSL/Docker → (固定URL) → px/alpaca (Windows上) → PACで振り分け → 社内プロキシ
  ```

- **WindowsローカルのプロキシをWSLから参照する場合**、NATモード(既定)では
  localhostが通じない
  - 案1: mirroredモードに切り替える(Windows 11 22H2以降)

    `%USERPROFILE%\.wslconfig`:

    ```ini
    [wsl2]
    networkingMode=mirrored
    ```

  - 案2: WindowsホストIPを動的取得して使う

    ```bash
    ip route show | grep -i default | awk '{ print $3}'
    ```

    ※ この場合、プロキシソフト側で「localhost以外からの接続」を許可する設定が必要
- **NO_PROXYの設定を忘れない**: 社内レジストリやlocalhostへのアクセスが
  プロキシ経由になって失敗するのを防ぐ。Dockerの内部ネットワーク
  (例: `172.17.0.0/16`)を加えることもある
- 認証付きプロキシのパスワードを設定ファイルに書く場合は
  `chmod 600` で権限を保護する

### WSL2全般

- プロジェクトファイルは `/mnt/c`(Windows側)ではなく **WSL内(`~/` 配下)に置く**
  (`/mnt/c` 経由のファイルI/Oは非常に遅い)
- WSL2のメモリ消費対策として `%USERPROFILE%\.wslconfig` で上限設定:

  ```ini
  [wsl2]
  memory=8GB
  processors=4
  ```

- Dockerイメージ等の肥大化対策: `docker system prune` で定期的に掃除。
  仮想ディスク(.vhdx)の圧縮には `wsl --manage Ubuntu --set-sparse true` や
  `Optimize-VHD` を利用
- VPN利用時にDNS解決が壊れる場合は `/etc/resolv.conf` の手動設定や
  `wsl.conf` の `generateResolvConf=false` で対処
- 設定変更が反映されない場合は `wsl --shutdown` で全体再起動するのが確実
